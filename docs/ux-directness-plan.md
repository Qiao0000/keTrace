# 日迹 Next 简单直达操作方案

## 1. 问题判断

当前 UI 已有 7 个主菜单和较完整功能，但直观度还不够，主要问题不是“页面少不够”，而是高频动作需要绕路。

典型摩擦：

- 创建项目要先进入任务页，再点项目按钮，再展开表单，再输入，再保存。
- 创建任务和归属项目是分开的两个动作。
- 时间块、任务、项目各自有表单，用户要先判断自己该点哪个按钮。
- 今日页主要是展示，快速记录能力还不够强。
- 论文/投稿也存在“先进入模块，再展开表单”的操作成本。

优化目标：

```text
常用动作 1 步直达
复杂对象渐进展开
项目/论文/投稿不阻塞记录
用户先记录，再整理
```

## 2. 核心原则

### 原则 1：任何页面都能快速新增

应用顶部固定一个全局快速输入框：

```text
输入任务、项目、论文进展或投稿动作...
```

用户不需要先想“我该去哪个页面”。

支持例子：

```text
修改 Figure 2 #论文 @明天 !高
新建项目 论文第三章
写引言 60min #论文第三章
投稿 Nature Communications @2026-07-15
返修回复审稿人 #投稿
```

输入后按 Enter 保存。

### 原则 2：项目是随用随建，不是前置流程

项目不应该必须先创建。用户输入：

```text
整理实验图 #第三章
```

如果 `第三章` 项目不存在，系统自动创建轻量项目，并把任务归进去。

项目创建降级成“打标签式创建”，不是单独工作流。

### 原则 3：主动作永远显眼

每页只保留一个最强主动作。

例如：

- 今日：快速记录
- 活动：开启/暂停采集
- 任务：新增任务
- 论文：记录论文进展
- 报告：生成报告
- 洞察：切换周期
- 设置：保存设置

其他动作放进二级区域或行内操作。

### 原则 4：表单不抢先出现

默认只给一个输入框。

用户需要细节时再点“展开更多”。

例如新增任务默认只需要：

```text
任务标题
```

可选项再展开：

```text
截止日期
优先级
项目
来源
备注
```

### 原则 5：列表项直接可操作

任务、章节、投稿卡片不只展示，还要能直接操作：

- 勾选完成。
- 一键排今天。
- 一键加 30 分钟时间块。
- 一键推进投稿阶段。
- 一键生成下一步任务。

不让用户为每个小改动打开编辑表单。

## 3. 新整体布局

保持 7 个主菜单不变：

```text
今日
活动
任务
论文
报告
洞察
设置
```

但增加一个全局操作栏：

```text
┌───────────────────────────────────────────────┐
│ 快速记录：输入任务、论文进展、投稿动作...      │
└───────────────────────────────────────────────┘
```

桌面布局：

```text
左侧菜单 + 顶部全局快速记录 + 当前页面主体
```

顶部区域固定：

```text
当前页标题
全局快速记录框
今日关键状态
```

## 4. 全局快速记录设计

### 输入规则

支持轻量语法：

```text
#项目
@日期
!优先级
30min / 1h
论文:
投稿:
```

示例解析：

```text
改论文摘要 #毕业论文 @今天 !高
```

生成：

- Task.title = 改论文摘要
- Project.name = 毕业论文
- dueDate = 今天
- priority = high

```text
论文: 写讨论部分 90min #毕业论文
```

生成：

- ThesisLog.type = 写作
- ThesisLog.minutes = 90
- ThesisLog.note = 写讨论部分
- 同时可选生成任务或时间块

```text
投稿: 补充 cover letter #NC @2026-07-15
```

生成：

- SubmissionLog 或 Task
- 如果 `NC` 投稿项目不存在，可提示“创建投稿项目？”

### 保存后反馈

输入框下方显示轻提示：

```text
已创建任务 · 已归入项目“毕业论文”
```

并提供撤销：

```text
撤销
```

## 5. 今日页改造

今日页应该成为真正的操作入口，而不是只看信息。

### 今日页结构

```text
今日
├─ 快速记录
├─ 今日主线
├─ 即将到期
├─ 最近活动
└─ 今日摘要
```

### 快速记录

放在第一屏最上方。

默认 placeholder：

```text
写下现在要做的事，例如：改摘要 #毕业论文 @今天 !高
```

右侧提供 3 个快捷按钮：

```text
任务
论文进展
投稿动作
```

点击按钮只是改变解析模式，不跳页。

### 今日主线

每条任务行内操作：

```text
开始
完成
排 30 分钟
排 60 分钟
```

不再要求用户进入任务页才能安排时间块。

### 即将到期

提醒项行内操作：

论文里程碑：

```text
生成任务
推迟
完成
```

投稿 deadline：

```text
生成下一步
进入投稿
推迟提醒
```

## 6. 任务页改造

任务页从“表单页”变成“执行台”。

### 新结构

```text
任务
├─ 快速新增任务
├─ 今日任务
├─ 时间块
├─ 项目筛选
└─ 全部任务
```

### 快速新增任务

只保留一行：

```text
[新增任务，支持 #项目 @日期 !高] [Enter]
```

不再默认展示完整表单。

### 项目创建优化

取消独立“+ 项目”主按钮，改为 3 种直达方式：

1. 输入 `#项目名` 自动创建。
2. 项目筛选下拉最底部提供 `+ 新建项目`。
3. 空项目状态显示 `创建第一个项目`。

### 任务行内操作

每个任务行展示：

```text
复选框 标题 项目 截止日期 优先级 [今天] [30m] [编辑]
```

其中：

- 复选框：完成/取消完成。
- 今天：设为今天任务。
- 30m：直接创建今天 30 分钟时间块。
- 编辑：才打开详细表单。

### 时间块创建优化

用户选中任务后可以直接点击：

```text
排今天 30m
排今天 60m
```

系统自动找一个默认开始时间：

- 如果当前时间在工作时间内，默认从当前下一个整点开始。
- 否则默认 09:00。

需要精确时间时再编辑时间块。

## 7. 论文页改造

论文页内部仍保留两个 Tab：

```text
论文
投稿
```

但默认入口应该是“记录进展”，不是维护资料。

### 论文 Tab

第一屏：

```text
记录论文进展
```

输入：

```text
写结果部分 90min 1200字
```

系统生成 ThesisLog。

同时提供可选按钮：

```text
同时生成任务
同时加入今日
```

### 论文资料维护

论文信息、章节、里程碑放在下方，作为管理区。

章节行内操作：

```text
进度 +10%
完成
记录 30m
生成任务
```

### 投稿 Tab

第一屏：

```text
记录投稿动作
```

输入：

```text
补 cover letter 30min #Nature
```

如果投稿项目不存在，出现轻提示：

```text
未找到 Nature 投稿项目，是否创建？
[创建] [仅作为任务]
```

投稿卡片行内操作：

```text
推进阶段
添加日志
生成下一步
设提醒
```

## 8. 报告页改造

报告页现在不要让用户先面对太多按钮。

### 新结构

```text
报告
├─ 一键生成
├─ 报告类型
├─ 预览
└─ 历史报告
```

第一屏主按钮：

```text
生成今日报告
```

旁边小切换：

```text
日 / 周 / 月
```

AI 摘要开关放在按钮下方，不作为主视觉。

历史报告放左侧或下方，不干扰主动作。

## 9. 活动页改造

活动页主动作：

```text
采集中 / 暂停采集
```

顶部显示状态：

```text
采集中 · 30 秒间隔 · 最近记录 12:36
```

如果权限失败，直接显示：

```text
采集可能未获得系统权限
[查看解决方法] [重试]
```

活动列表行内操作：

```text
标记为工作
标记为娱乐
忽略此应用
生成任务
```

第一版可以先只做展示，第二版再做分类操作。

## 10. 洞察页改造

洞察页不做复杂控制。

顶部只保留：

```text
7 天 / 30 天
```

图表按固定顺序：

1. 工作时长趋势。
2. 应用占比。
3. 任务完成率。
4. 论文投入。
5. 投稿阶段。

每个图表空状态都给一个直达动作：

```text
暂无论文投入记录
[记录论文进展]
```

## 11. 设置页改造

设置页减少“立即保存”的频繁触发。

当前一些输入每次 onChange 都保存，容易造成频繁写文件。

建议：

- 开关类：点击即保存。
- 文本类：失焦保存，或统一“保存”。
- API Key：输入后点击“保存并测试”。
- 采集间隔：使用 stepper 或 slider，停止调整后保存。

设置页增加数据目录直达：

```text
打开数据目录
打开报告目录
创建备份
恢复备份
```

## 12. 命令面板

加入 `Cmd/Ctrl + K` 命令面板。

支持：

```text
新增任务
新增项目
记录论文进展
新增投稿项目
生成日报
打开设置
打开数据目录
开启采集
暂停采集
```

命令面板是“高级直达”，全局快速记录是“日常直达”。

## 13. 关键工作流对比

### 创建项目

现在：

```text
任务页 -> +项目 -> 输入名称 -> 保存 -> 新增任务 -> 选择项目
```

优化后：

```text
输入：整理第三章 #毕业论文
Enter
```

项目不存在时自动创建。

### 创建今日任务

现在：

```text
任务页 -> +新增任务 -> 输入 -> 选日期/项目 -> 保存
```

优化后：

```text
今日页输入：改摘要 @今天 #毕业论文
Enter
```

### 安排时间块

现在：

```text
任务页 -> +时间块 -> 填标题 -> 填开始 -> 填结束 -> 选任务 -> 保存
```

优化后：

```text
任务行点击：排 60m
```

需要精确时间时再编辑。

### 记录论文进展

现在：

```text
论文页 -> +推进日志 -> 填日期/类型/分钟/字数/备注 -> 保存
```

优化后：

```text
输入：写讨论 90min 800字
Enter
```

### 投稿推进

现在：

```text
论文页 -> 投稿 Tab -> 找项目 -> 添加日志/推进阶段
```

优化后：

```text
输入：投稿: 回复审稿人 45min #NC
Enter
```

## 14. 实施顺序

### 第一阶段：快速记录核心

目标：让“新增任务/项目”一步完成。

任务：

1. 增加全局快速记录栏。
2. 实现 `#项目` 自动创建。
3. 实现 `@今天/@明天/@日期` 解析。
4. 实现 `!高/!普通/!低` 解析。
5. 保存后显示反馈和撤销。

### 第二阶段：任务页直达

目标：任务无需展开表单即可执行。

任务：

1. 任务页顶部改成一行快速新增。
2. 项目按钮降级为下拉中的“新建项目”。
3. 任务行增加完成、今天、30m、60m 操作。
4. 时间块自动生成默认时间。

### 第三阶段：今日页变成操作台

目标：用户每天主要在今日页工作。

任务：

1. 今日页第一屏放快速记录。
2. 今日主线支持完成/开始/排时间。
3. 论文/投稿提醒支持生成任务。
4. 最近活动支持刷新。

### 第四阶段：论文/投稿直达

目标：记录论文和投稿不用填大表单。

任务：

1. 论文 Tab 顶部加“记录论文进展”输入。
2. 投稿 Tab 顶部加“记录投稿动作”输入。
3. 章节行加 `记录 30m` 和 `生成任务`。
4. 投稿卡加 `推进阶段` 和 `生成下一步`。

### 第五阶段：命令面板

目标：键盘用户快速直达。

任务：

1. 实现 `Cmd/Ctrl + K`。
2. 支持页面跳转。
3. 支持常用创建命令。
4. 支持打开目录、生成报告、开关采集。

## 15. 验收标准

优化后，高频动作必须满足：

```text
新增任务：1 次输入 + Enter
创建项目：随任务自动创建
安排时间块：任务行 1 次点击
记录论文进展：1 次输入 + Enter
记录投稿动作：1 次输入 + Enter
生成日报：1 次点击
切换洞察周期：1 次点击
```

页面直观标准：

- 用户打开应用第一眼就知道可以在哪里输入。
- 每页只有一个最明显主动作。
- 没有数据时，空状态提供直达动作。
- 创建项目不再是独立前置流程。
- 复杂表单默认隐藏，只在编辑详情时出现。

## 16. 最终体验目标

日迹 Next 不应该像一个“管理后台”，而应该像一个每天打开就能直接工作的桌面工具。

核心体验：

```text
想到什么 -> 直接输入
系统帮你归类
需要细节时再整理
报告和洞察自动生成
```

这会比单纯增加按钮或减少菜单更有效，也能保留现有功能深度。

## 17. 参考项目 UI 吸收方向

UI 美化不应该只是换颜色，而是把参考项目里已经验证过的体验优势吸收进来。

参考来源：

```text
/Users/mymac/Desktop/日迹/templates/report_template.html
/Users/mymac/Desktop/日迹/参考项目/博士工作台 2.0.html
/Users/mymac/Desktop/日迹/参考项目/phd_master_system-0.2.7/phd_master_system-0.2.7/app/index.html
```

### 日迹旧版可继承的优点

旧日迹区域 UI 的优点：

- 信息密度适中。
- 侧边栏清晰。
- 活动时间线直观。
- 热力图和应用时长适合“看一天怎么过去了”。
- 报告页有安静、克制的数据仪表盘气质。
- 设置页分组清楚。

应吸收：

- 活动页继续保留“时间线 + 应用时长 + 热力图”的组合。
- 报告预览保持文档感，不做花哨卡片。
- 洞察页用简洁统计卡和图表，不做过多装饰。
- 设置页保持分组式结构。

应避免：

- 旧日迹部分区域偏报告查看，操作入口弱。
- 一些按钮只是展示型，没有形成快速操作。
- 页面跳转和生成报告的反馈还可以更明确。

### 博士工作台可继承的优点

博士工作台的优点：

- “今日执行舱”概念很适合首页。
- 任务、论文、投稿之间有工作流联动。
- 论文/投稿模块结构完整。
- 提醒中心和今日主线有实际使用价值。
- 数据洞察能覆盖执行、科研、复盘等维度。
- 行内状态标签、阶段看板和推进日志很适合学术工作。

应吸收：

- 首页采用“今日执行舱”结构。
- 论文页保留论文信息、章节、里程碑、推进日志。
- 投稿页保留投稿阶段看板。
- 今日页聚合论文/投稿提醒。
- 洞察页加入论文投入、投稿阶段、任务完成率。
- 卡片内使用状态标签和进度条，让页面一眼可扫。

应避免：

- 菜单过多。
- 表单过多。
- 每个模块都像完整后台，日常操作路径太长。
- 颜色过于活泼时会削弱“日迹”的安静工具气质。

## 18. 新 UI 视觉基调

日迹 Next 的 UI 应该是：

```text
安静
清楚
可扫视
少跳转
高频动作靠前
学术工作不压迫
```

关键词：

```text
执行舱
时间证据
轻量任务
学术推进
报告输出
本地隐私
```

整体视觉不要走营销页、游戏化、强装饰路线。它应该像一个可靠的桌面工具，而不是网页后台模板。

## 19. 布局美化方案

### 整体布局

保留左侧窄侧边栏，但压低视觉噪音。

建议：

```text
左侧：主菜单
顶部：页面标题 + 全局快速记录
主体：当前页面
右侧：可选上下文面板，第一版不默认使用
```

布局示意：

```text
┌────────────┬────────────────────────────────────────────┐
│ 日迹 Next  │ 今日                                      │
│            │ [快速记录：改摘要 #毕业论文 @今天 !高]    │
│ 今日       ├────────────────────────────────────────────┤
│ 活动       │ 今日主线        最近活动                  │
│ 任务       │ 即将到期        论文/投稿提醒              │
│ 论文       │ 今日摘要                                  │
│ 报告       │                                            │
│ 洞察       │                                            │
│ 设置       │                                            │
└────────────┴────────────────────────────────────────────┘
```

### 侧边栏

参考旧日迹的清晰侧边栏，但减少装饰。

规范：

- 宽度 208-228px。
- 菜单只保留 7 项。
- 当前项使用实色背景。
- 非当前项只使用轻 hover。
- 图标统一，不使用混杂符号。
- 底部显示本地状态，例如：

```text
本地优先 · 采集中
```

建议图标：

```text
今日：calendar/check
活动：activity/clock
任务：list-check
论文：book-open
报告：file-text
洞察：bar-chart
设置：settings
```

后续如果引入图标库，优先 lucide。

### 顶部快速记录区

这是美化和操作逻辑的核心。

样式：

- 不是普通 input，要像“命令输入条”。
- 放在每页标题下方。
- 高度 44-52px。
- 圆角 10-12px。
- 左侧有输入图标。
- 右侧有解析提示，如 `#项目 @日期 !优先级`。
- 保存成功后在输入框下方显示轻量反馈。

示例：

```text
┌─────────────────────────────────────────────────────┐
│ +  改摘要 #毕业论文 @今天 !高        #项目 @日期 !高 │
└─────────────────────────────────────────────────────┘
```

### 卡片

参考博士工作台的模块化卡片，但减少圆角和饱和色。

规范：

- 卡片圆角 8-10px。
- 边框优先，阴影很轻。
- 卡片标题小而清楚。
- 卡片内主操作放右上角。
- 不做卡片套卡片。

卡片层级：

```text
一级：页面主要区域
二级：列表项/任务项
弹层：编辑详情
```

避免：

- 大面积彩色背景。
- 多层嵌套卡片。
- 每个统计都做成大卡片，导致页面很散。

### 状态标签

参考博士工作台的状态标签，但统一语义。

任务：

```text
todo      灰
doing     蓝
done      绿
blocked   红/橙
high      红色边框
```

论文章节：

```text
待写       灰
起草中     蓝
修改中     橙
完成       绿
```

投稿阶段：

```text
写作中     蓝
待投稿     紫
已投稿     青
审稿中     橙
返修中     红
已接收     绿
搁置/拒稿  灰
```

标签样式：

- 小字号。
- 背景浅色。
- 文字深色。
- 不使用过饱和底色。

## 23. 美化实施顺序

不要一次性重写所有页面。

推荐顺序：

### 第一步：统一视觉 Token

- 调整 `global.css` 中颜色、圆角、间距。
- 降低强紫色占比。
- 统一标签颜色。
- 调整按钮和输入框尺寸。

### 第二步：加入全局快速记录

- 顶部加入 `QuickCaptureBar`。

### 第三步：任务页行内操作

- 把表单降级。
- 任务行变成可直接操作。
- 项目自动创建。

### 第四步：今日页执行舱

- 参考博士工作台今日执行舱。
- 第一屏展示今日主线、提醒、最近活动。
- 加行内操作。

### 第五步：论文/投稿重排

- 顶部加入进展输入。
- 章节进度条优化。

- 活动时间线接近日迹旧版。
- 报告预览接近日迹旧版文档感。
- 洞察图表低饱和。

## 24. UI 验收标准

视觉验收：

- 第一屏能看到快速记录入口。
- 7 个菜单清楚但不抢内容。
- 每页只有一个主动作。
- 空状态都有下一步动作。
- 表单不默认占据页面主体。
- 任务和论文数据可以一眼扫出状态。
- 活动页能直观看到“今天做了什么”。
- 报告页像文档，不像后台。

操作验收：

- 创建项目不超过 1 次输入。
- 创建任务不超过 1 次输入。
- 任务排时间不超过 1 次点击。
- 记录论文进展不超过 1 次输入。
- 推进投稿不超过 1 次点击或 1 次输入。

情绪验收：

- 页面不应让用户感觉“又多了一个要维护的系统”。
- 应该让用户感觉“我只要随手记，它会帮我整理”。
